check/1

check(X1) -> any()

This function is called according to the interval defined in the overload specification. In this example the check compares the amount of free memory of the system to a threshold. If the amount of free memory is less than the threshold, {suspend, Reason} is returned and all trace patterns are deactivated by Onviso. In the other case tracing will continue as normal.

It is possible to pass data to the check function. It accepts one argument and it is the return value from the 'start/0' function which is passed. Note that it is possible to pass arguments to the start function from the overload specification, i.e the start arity doesn't have to be 0.

Note: this example only works on *nix like systems with the free script installed.

  free | grep -i mem | awk '{print $4}'

start/0

start() -> any()

Initiates the overload handler process. In this case it actually only returns ok since no set-up is required.

What is considered an overload situation depends on the type of system you are running. That is why it has to be defined outside the scope of Onviso.

stop/0

stop() -> any()

Cleans up the overload handler process.
